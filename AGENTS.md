# AGENTS.md

## Project Facts

- Bun is the source-of-truth package manager/runtime here; `bun.lock` is committed and scripts use Bun-only APIs such as `Bun.write` and `bun build --compile`.
- CLI entrypoint is `src/index.ts`; the published `bin` points at generated `dist/index.js`.
- Core execution is in `src/client.ts`: it builds concurrent `listr2` tasks over deduped positional URLs, uses `@vot.js/node`/`@vot.js/shared`, and writes either downloads or preview links.
- CLI options are defined in `src/resources/schema.ts`; help text is separate in `src/resources/messages.ts`, so option changes usually need both files updated.

## Generated Files

- `src/config.ts` is generated by `bun run update:config` from `package.json` version plus `@vot.js/shared/config`; do not hand-edit it except to inspect output.
- `src/types/schema.ts` is generated by `bun run update:schema` from `src/resources/schema.ts`; edit the schema source, then regenerate.
- `bun run build`, `bun run build:bun`, and `bun run compile` run `update:all` before building.

## Commands

- Install deps: `bun install`.
- Regenerate config and schema only: `bun run update:all`.
- Build npm/Node output in `dist/`: `bun run build:bun` or `bun run build`.
- Cross-compile standalone binaries into `build/`: `bun run compile`.
- Lint: `bun run lint`; auto-fix: `bun run lint:fix`.
- Format check/write with oxfmt: `bun run fmt:check` / `bun run fmt`.
- There are currently no test files or `test` script; verify changes with lint, format check, and the relevant build command.

## Tooling Quirks

- Pre-commit is Lefthook and runs `bunx oxfmt --no-error-on-unmatched-pattern` plus `bun run lint` on staged JS/TS files.
- Formatting/linting config used by scripts is `.oxfmtrc.json` and `.oxlintrc.json`; `eslint.config.js` and `.prettierrc.json` exist but package scripts do not call them.
- `dist/`, `build/`, `node_modules/`, `debian/`, `*.mp3`, and `*.tmp` are ignored; avoid treating existing local media/build artifacts as source.
- README notes Bun cross-compilation can fail on Windows with `Failed to move cross-compiled bun binary into cache directory`; use WSL or move the repo to the C: drive for that case.

## Release/CI

- CI build installs with Bun and runs `bun build:bun`; compile CI runs `bun compile` on Ubuntu.
- Release workflow only triggers on pushes to `master` or `2.0.0` and creates a draft GitHub release from compiled artifacts.
